# 像素地牢 - 使用说明

## 环境要求

- **JDK 17 或更高版本**（确保命令行输入 `java -version` 能正常显示）
- **Windows 10 或更高版本**
- 任意现代浏览器（Edge、Chrome、Firefox 均可）

---

## 一、快速开始（日常游玩）

### 方式 A：终端启动（推荐）

打开终端（cmd 或 PowerShell），执行：

```bash
cd /d "C:\Users\张坤源\Desktop\像素地牢（测试）\backend\target"
java -jar dungeon-backend-1.0.0.jar
```

看到 `Started DungeonApplication` 后，浏览器访问 **`http://localhost:8080`** 即可游玩。

> 下次启动只需要输入 `java -jar dungeon-backend-1.0.0.jar` 然后按 ↑ 键调出历史命令。

**关闭服务器：** 在终端窗口按 **`Ctrl + C`** 即可正常停止。

> `Ctrl + C` 会让 Spring Boot 正常断开数据库连接再退出，比强制杀进程更安全。

### 方式 B：一键启动脚本

双击项目根目录下的 **`start-game.bat`** 启动，双击 **`stop-server.bat`** 关闭。

> 不玩的时候关不关后端都行，后台几乎不耗资源。

## 二、手机端游玩

### 连接方式

1. 电脑和手机连接**同一个 WiFi**
2. 启动后端（双击 `start-game.bat` 或在终端执行 `java -jar`）
3. 在电脑上打开终端输入 `ipconfig`，找到 IPv4 地址
4. 手机浏览器访问 `http://<电脑IP>:8080`

### 手机触控操作

横屏模式下（旋转手机为横屏）：

| 操作 | 方式 |
|------|------|
| 移动 | 左侧白色摇杆，拖拽控制方向 |
| 瞄准射击 | 右侧红色摇杆，拖拽瞄准，持续按住连射 |
| 翻滚 | 右下角红色"翻"按钮 |
| 交互（开门/开箱） | 底部中央 F 按钮 |
| 切换武器 | 底部中央 Q 按钮 |
| 暂停 | 右上角 \|\| 按钮 |

### 防火墙设置

如果手机无法访问，在电脑上以**管理员身份**打开终端，执行：

```
netsh advfirewall firewall add rule name="像素地牢 8080" dir=in action=allow protocol=TCP localport=8080
```

## 三、分享游戏给朋友

只需发送 `backend\target` 目录下的这两个文件：

1. **`dungeon-backend-1.0.0.jar`**（游戏本体，约 46 MB）
2. **`启动游戏.bat`**（一键启动脚本）

对方需要：
- 安装 JDK 17+（https://www.oracle.com/java/technologies/downloads/）
- 把两个文件放在同一个文件夹里
- 双击 `启动游戏.bat`
- 浏览器自动打开 `http://localhost:8080` 即可开始玩

> 所有数据（用户、存档、日志）自动保存在 `C:\Users\<用户名>\dungeon_data\` 目录下，随身带走只需复制该文件夹。

## 四、修改代码后重新构建

如果你修改了前端代码（`index.html`、`game.js`、`api.js`）或后端代码：

1. 先停止正在运行的服务（`Ctrl + C` 或双击 `stop-server.bat`）
2. 打开终端（cmd 或 PowerShell），进入 `backend` 目录
3. 执行构建命令：
   ```
   mvnw clean package -DskipTests
   ```
4. 构建成功后重新启动

> **重要**：修改前端文件后，必须使用 `clean package`（不能只用 `package`），否则 Maven 增量编译缓存可能导致修改后的文件没有打进 JAR。

## 五、查看 H2 数据库中的数据

### 步骤 1：确保后端正在运行

先启动后端（终端 `java -jar` 或双击 `start-game.bat`）。

### 步骤 2：打开 H2 控制台

浏览器访问：**`http://localhost:8080/h2-console`**

### 步骤 3：登录数据库

在登录页填入以下信息：

| 字段 | 填写内容 |
|------|----------|
| JDBC URL | `jdbc:h2:file:~/dungeon_data/dungeon_db` |
| 用户名 | `sa` |
| 密码 | （留空） |

点击 **Connect** 按钮。

### 步骤 4：查询数据

连接成功后，左侧会显示三张表：

| 表名 | 存储内容 |
|------|----------|
| `USERS` | 用户账号（密码为 BCrypt 加密） |
| `SAVE_DATA` | 大厅存档（宝石、升级、统计数据） |
| `RUN_SAVE_DATA` | 单局中途存档（当前楼层、击杀数等） |

在右侧输入框执行查询，点击 **Run** 按钮即可查看结果。

#### 基础查询

```sql
-- 查看所有注册用户
SELECT * FROM USERS;

-- 查看所有大厅存档
SELECT * FROM SAVE_DATA;

-- 查看正在进行的单局存档
SELECT * FROM RUN_SAVE_DATA;
```

#### 查询特定用户

```sql
-- 按用户名查用户信息（替换 '你要查的用户名' 为实际名称）
SELECT * FROM USERS WHERE username = '你要查的用户名';

-- 按用户 ID 查用户信息（替换 1 为实际 ID）
SELECT * FROM USERS WHERE id = 1;
```

#### 查询特定用户的存档

```sql
-- 查某用户的大厅存档（宝石、升级等级、战绩）
SELECT u.username, s.*
FROM SAVE_DATA s
JOIN USERS u ON s.user_id = u.id
WHERE u.username = '你要查的用户名';

-- 查某用户的单局存档（当前楼层、击杀、金币等）
SELECT u.username, r.*
FROM RUN_SAVE_DATA r
JOIN USERS u ON r.user_id = u.id
WHERE u.username = '你要查的用户名';
```

#### 一键查看某用户全部数据

```sql
SELECT u.id, u.username, u.created_at,
       s.gems, s.upgrades_json, s.stats_json,
       r.has_run, r.floor_num, r.kills, r.coins, r.gems_this_run
FROM USERS u
LEFT JOIN SAVE_DATA s ON s.user_id = u.id
LEFT JOIN RUN_SAVE_DATA r ON r.user_id = u.id
WHERE u.username = '你要查的用户名';
```

## 七、常见问题

**Q: 启动时提示"未检测到 Java"**

需要安装 JDK 17+，下载地址：https://www.oracle.com/java/technologies/downloads/

安装后确保在命令行输入 `java -version` 能正常显示版本号。

**Q: 端口 8080 已被占用**

说明后端已在运行，无需重复启动。直接访问 `http://localhost:8080` 即可。

如果确认没有运行，双击 `stop-server.bat` 清理残留进程后重试。

**Q: 如何关闭后端服务**

在运行后端的终端窗口按 **`Ctrl + C`** 即可。如果使用了 bat 脚本启动（后台无窗口），则双击 `stop-server.bat`。

**Q: H2 控制台连接失败**

确保后端先启动成功，且 JDBC URL 填写正确（注意是 `~/dungeon_data/dungeon_db`，不是本机的绝对路径）。

**Q: 登录时提示"用户不存在"**

先注册新账号，点击登录面板下方的"还没有账号？点击注册"切换到注册模式。

**Q: 分享游戏时对方无法启动**

确保对方也安装了 JDK 17+，并把 `.jar` 文件和 `.bat` 文件放在同一目录下。

**Q: 页面能打开但所有按钮点击无反应（登录/注册等都无法操作）**

通常是修改前端代码后没有重新构建，或 JavaScript 有语法错误。解决方法：

```
cd backend
mvnw clean package -DskipTests
```

执行完成后再启动游戏。

**Q: 双击 start-game.bat 后后端没有启动**

1. 直接用终端启动看具体报错（推荐）：
   ```bash
   cd /d "C:\Users\张坤源\Desktop\像素地牢（测试）\backend\target"
   java -jar dungeon-backend-1.0.0.jar
   ```
   终端会显示完整的 Spring Boot 日志，任何错误一目了然。
2. 也可以查看日志文件：`C:\Users\<你的用户名>\dungeon_data\backend.log`
3. 如果端口被占用，双击 `stop-server.bat` 清理残留进程后重试

**Q: 后端启动后立即退出**

查看 `C:\Users\<你的用户名>\dungeon_data\backend.log`，常见原因：
- 数据库文件损坏 → 删除 `C:\Users\<你的用户名>\dungeon_data\` 目录后重试（注意备份存档）
- 端口冲突 → 双击 `stop-server.bat` 清理残留进程
