# 像素地牢（元气地牢）—— 项目制作提示词

## 项目概述

制作一个名为"元气地牢"的像素风格 Roguelike 动作射击游戏。玩家在随机生成的地牢中探索，击杀敌人、拾取武器、选择天赋，最终击败 Boss 通关。支持桌面端（键鼠）和手机端（触控横屏），支持账号注册登录和云端存档。

### 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Spring Boot 3.2.0，Java 17 |
| 数据库 | H2 File Database（文件模式，自动建表） |
| ORM | Spring Data JPA + Hibernate |
| 安全 | Spring Security + JWT（jjwt 0.12.3） |
| 前端 | 纯 JavaScript + HTML5 Canvas 2D，无框架 |
| 构建 | Maven（spring-boot-maven-plugin） |

### 2.3 数据库实体

**User 表（users）**
```java
@Entity @Table(name = "users")
public class User {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(unique = true, nullable = false, length = 32)
    private String username;
    
    @Column(name = "password_hash", nullable = false, length = 255)
    private String passwordHash;
    
    private LocalDateTime createdAt;
    // created_at 在创建时自动设为 LocalDateTime.now()
}
```

**SaveData 表（save_data）—— 大厅永久存档**
```java
@Entity @Table(name = "save_data")
public class SaveData {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @OneToOne
    @JoinColumn(name = "user_id", unique = true, nullable = false)
    private User user;
    
    private int gems = 0;
    
    @Column(name = "upgrades_json", columnDefinition = "TEXT")
    private String upgradesJson;  // 默认: {"maxHp":0,"maxShield":0,"maxEnergy":0,"cooldownReduct":0,"chestQuality":0}
    
    @Column(name = "stats_json", columnDefinition = "TEXT")
    private String statsJson;     // 默认: {"totalRuns":0,"totalKills":0,"bestFloor":0}
    
    private LocalDateTime updatedAt;
}
```

**RunSaveData 表（run_save_data）—— 单局中途存档**
```java
@Entity @Table(name = "run_save_data")
public class RunSaveData {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @OneToOne
    @JoinColumn(name = "user_id", unique = true, nullable = false)
    private User user;
    
    private boolean hasRun = false;
    private int floorNum;
    private int kills;
    private int coins;
    private int gemsThisRun;
    
    @Column(name = "player_json", columnDefinition = "TEXT")
    private String playerJson;
    
    @Column(name = "rooms_json", columnDefinition = "TEXT")
    private String roomsJson;
    
    private int currentRoomIndex;
    private LocalDateTime updatedAt;
}
```

### 2.6 Service 层

**JwtService：**
- 从配置文件读取 `jwt.secret` 和 `jwt.expiration-ms`
- 使用 `Keys.hmacShaKeyFor()` 创建 HMAC-SHA 密钥
- `generateToken(username)` → 生成 JWT，sub 为 username，包含 issuedAt 和 expiration
- `extractUsername(token)` → 解析 token 获取 username
- `validateToken(token)` → 验证 token 有效性

**AuthService：**
- 注入 UserRepository、JwtService、BCryptPasswordEncoder（在 SecurityConfig 中定义为 @Bean）
- `register(username, password)` → 检查重复 → BCrypt 加密密码 → 保存用户 → 返回 JWT
- `login(username, password)` → 查找用户 → 验证密码 → 返回 JWT

**SaveService：**
- 注入 SaveDataRepository、RunSaveDataRepository
- 所有写入操作加 `@Transactional`
- `loadSave(User)` / `saveSave(User, DTO)` → 读取/更新大厅存档
- `loadRunSave(User)` / `saveRunSave(User, DTO)` / `deleteRunSave(User)` → 读取/更新/删除单局存档
- 首次读取时自动创建默认存档记录

### 2.7 Controller 层

**AuthController (`/api/auth`)：**
```
POST /api/auth/register  → @Valid RegisterRequest → AuthResponse
POST /api/auth/login     → @Valid LoginRequest → AuthResponse
```
两个接口均添加 `@CrossOrigin(origins = "*")`，参数使用 `@Valid` 验证。

**SaveController (`/api/save`)：**
```
GET    /api/save/load          → 加载大厅存档
POST   /api/save/save           → 保存大厅存档
GET    /api/save/run/load       → 加载单局存档
POST   /api/save/run/save       → 保存单局存档
POST   /api/save/run/delete     → 删除单局存档（死亡/通关后）
```
- 所有接口通过私有方法 `getUserFromToken(authHeader)` 从 Authorization header 中提取 Bearer token
- 解析 JWT → 查询用户 → 执行操作
- 返回 JSON

### 2.8 安全和跨域配置

**SecurityConfig：**
```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) {
    http.csrf(csrf -> csrf.disable())
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
            .requestMatchers("/h2-console/**").permitAll()
            .anyRequest().permitAll()
        )
        .headers(headers -> headers.frameOptions(frame -> frame.disable()));
    return http.build();
}

@Bean
public BCryptPasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```
注意：不创建 JWT filter，认证逻辑在 SaveController 中手动处理。

**WebConfig：**

```java
public void addCorsMappings(CorsRegistry registry) {
    registry.addMapping("/api/**")
        .allowedOrigins("*")
        .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
        .allowedHeaders("*");
}
```

### 3.3 game.js —— 游戏引擎（核心，约 3400 行）

#### 3.3.1 存档系统

- 本地存存档 key：`yuanqi_dungeon_save`（localStorage）
- `globalSave` 对象包含：`{ gems, upgrades: {maxHp, maxShield, maxEnergy, cooldownReduct, chestQuality}, stats: {totalRuns, totalKills, bestFloor} }`
- 加载优先级：先读本地，若已登录则读取云端并合并
- 保存策略：本地 + 云端双写，游戏中每15秒自动存档

#### 3.3.2 Canvas 和输入

- **Canvas DPI 适配：** 使用 `devicePixelRatio`，`canvas.width = screenW * dpr`
- **渲染统一使用 CSS 坐标：** 每帧 `ctx.setTransform(dpr, 0, 0, dpr, 0, 0)`
- **InputManager（键盘+鼠标）：**
  - WASD / 方向键 → 移动
  - 鼠标左键 → 朝鼠标方向射击
  - 鼠标右键 → 翻滚（朝鼠标方向）
  - Q → 切换武器
  - F → 交互（开箱/进门/使用传送门）
  - ESC → 暂停/关闭面板
- **TouchInputManager（手机触控）：**
  - 设备检测：`'ontouchstart' in window || navigator.maxTouchPoints > 0`
  - 左侧 35% 屏幕 → 固定位置白色虚拟摇杆（移动）
  - 右侧 65% 屏幕 → 固定位置红色虚拟摇杆（瞄准射击，持续按住连射）
  - 左下按钮 → 翻滚、交互（F）、切换武器（Q，300ms防抖）
  - 右上按钮 → 暂停
  - 竖屏检测 → 显示旋转提示

#### 3.3.3 资源加载

使用 `AssetLoader` 类读取所有精灵图片，显示加载进度条。支持的精灵：
- **玩家：** Knight（idle, run, attack1, hurt, death）
- **敌人：** Slime, Skeleton, Mushroom, Wizard（idle, walk, attack, hurt, death），Boss（额外有 fly）
- **道具：** chest_open, chest_closed, barrel, stone_column, crate, table, chair, candle, carpet, bookcase, display_case, hay_bales, fence
- **地板：** tilemap 纹理

| kill_energy | 击杀回能 | 资源 | 稀有 | 击杀回复15能量 |
| coin_magnet | 吸金体质 | 资源 | 稀有 | +60 拾取范围 |

**升级机制：**
- 初始 expToNext = 8，每级 ×1.4
- 升级时暂停游戏，弹出3个随机天赋选项（从未获得的天赋中抽取）
- 品质概率：普通 85%、稀有 15%

#### 3.3.9 房间系统

**房间尺寸：** 720×480 像素
**墙壁厚度：** 20px
**房间类型：**
- `start` — 出生点，低围栏装饰
- `normal` — 普通战斗房，木桶/板条箱/椅子装饰
- `elite` — 精英房，石柱/烛台/桌子装饰，难度更高
- `boss` — Boss房，石柱/烛台/地毯，击杀后生成传送门
- `chest` — 宝箱房，书架/展示柜，含宝箱
- `shop` — 商店，长桌/椅子/烛台

**战斗机制：**
- 首次进入非start房间 → 关闭所有门 → 生成敌人 → 开始战斗
- 全部敌人死亡 → 打开门 → 标记已清理
- Boss房间全部敌人死亡 → 生成蓝色传送门（去下一层）

**门和过渡：**
- 每个房间有上下左右4个门位（根据邻居决定是否开启）
- 玩家靠近门边缘 → 0.3秒渐隐过渡 → 切换到相邻房间

#### 3.3.10 地牢生成算法

- 5×5 网格
- 从中心 (2,2) 开始随机游走
- 每次选择一个随机的未占用相邻格子
- 如果无路可走，随机选择一个有可用邻居的已存在房间继续
- 共生成 7-12 个房间
- 离出生点最远的房间设为 Boss 房
- 15% 精英房，10% 宝箱房，5% 商店
- 至少保证 1 个宝箱房和 1 个商店
- 共 3 层，第 4 层触发胜利

#### 3.3.11 大厅升级

5个升级项，用宝石购买：
- 最大生命 (maxHp)：每级 +1（0-5级）
- 最大护盾 (maxShield)：每级 +1（0-5级）
- 最大能量 (maxEnergy)：每级 +20（0-5级）
- 冷却缩减 (cooldownReduct)：翻滚 CD 每级 -12%（0-5级）
- 宝箱品质 (chestQuality)：提高好掉落概率（0-3级）

#### 3.3.12 商店（局内）

操作方式：F键与商人交互 → 进入商店界面

| 商品 | 价格（金币） | 效果 |
|------|------------|------|
| 治疗 | 30 | 回复2HP |
| 护盾恢复 | 20 | 补满护盾 |
| 能量恢复 | 25 | 补满能量 |
| 随机武器 | 50 | 获得一把随机品质武器 |

#### 3.3.13 HUD 显示

- **HP条：** 红色，左上角
- **护盾条：** 蓝色，HP条下方
- **能量条：** 黄色，护盾条下方
- **经验条：** 绿色，能量条下方
- **金币/宝石/击杀/楼层数：** 右上角
- **武器栏：** 下方中央，显示当前装备4个武器图标
- **翻滚冷却指示器：** 右下角
- **小地图：** 右侧，5×5网格，颜色编码（当前=红，已清理=绿，已访问=橙，未访问=深灰，Boss/宝箱/商店=特殊标记）
- **Boss接近警告：** Boss房间相邻时显示红色脉冲

#### 3.3.14 摄像机

- 缩放比例：`min(screenW/720, screenH/480) × 0.9`
- 平滑跟随玩家（lerp 8×速度）
- 限定在房间边界内（40px边距）
- 受伤时屏幕震动（0.15s），Boss受伤震0.2s

#### 3.3.15 粒子特效

- **ObjectPool** 管理子弹、粒子、浮动文字的对象池
- **ParticleSystem：** 位置、速度、生命、大小、颜色、淡出
- **FloatingText：** 伤害数字、拾取提示，向上漂移 + 淡出
- **子弹碰撞：** 与墙壁碰撞消失，与敌人/玩家碰撞造成伤害（支持穿透）

#### 3.3.16 结算

- **胜利：** 通过第3层传送门 → 结算界面
- **死亡：** HP归零 → 0.5s死亡动画 → 结算界面
- **宝石获得：** 基础 = 击杀×2 + 楼层×10 + 50(通关奖励)，总计 = 基础 + 局内拾取
- 更新总计数据和最佳楼层
- 删除云端单局存档

## 六、关键设计决策

1. **前后端不分离** — 前端文件（index.html、game.js、api.js）放在 `src/main/resources/static/` 下，Spring Boot 自动作为静态资源提供服务
2. **Canvas 2D 渲染** — 不使用任何游戏引擎或框架，所有渲染通过原生 Canvas API 完成
3. **对象池** — 子弹、粒子、浮动文字使用对象池减少 GC 压力
4. **双存档系统** — 大厅存档（永久升级）和单局存档（中途存档）分离
5. **认证自管理** — 不使用 Spring Security 的 JWT Filter 链，而是在 Controller 中手动解析 Token 获取用户
6. **同时支持桌面和手机** — 通过设备检测自动适配输入方式（键鼠 vs 触控摇杆）和 UI（横屏提示）
7. **H2 文件数据库** — 数据存储在 `C:\Users\<用户名>\dungeon_data\` 目录，方便备份迁移
