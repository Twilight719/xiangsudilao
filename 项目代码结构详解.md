## 一、项目整体结构

```
像素地牢（测试）/
├── backend/                                    ← 后端 Spring Boot 项目
│   ├── pom.xml                                 ← Maven 依赖配置
│   ├── mvnw / mvnw.cmd                         ← Maven Wrapper（无需安装Maven）
│   └── src/main/
│       ├── java/com/dungeon/                   ← 所有 Java 源码
│       │   ├── DungeonApplication.java
│       │   ├── config/         (2个文件)
│       │   ├── controller/     (2个文件)
│       │   ├── dto/            (5个文件)
│       │   ├── exception/      (1个文件)
│       │   ├── model/          (3个文件)
│       │   ├── repository/     (3个文件)
│       │   └── service/        (3个文件)
│       └── resources/
│           ├── application.properties          ← 应用配置文件
│           └── static/                         ← 前端静态文件（Spring Boot直接提供服务）
│               ├── index.html
│               ├── game.js
│               └── api.js
├── assets/                                     ← 游戏美术资源
├── props/                                      ← 场景道具贴图
├── tilemaps/                                   ← 地板纹理贴图
├── start-game.bat                              ← 一键启动脚本
├── stop-server.bat                             ← 停止服务脚本
└── 使用说明.md                                  ← 用户使用文档
```

**项目类型：** Spring Boot 3.2.0 + Java 17 + H2 文件数据库  
**前端：** 纯 JavaScript（无框架）+ HTML5 Canvas 2D  
**构建：** Maven，输出 fat JAR（约46MB），包含全部前端和依赖

## 二、后端 Java 代码

> 所有 Java 文件位于：`backend/src/main/java/com/dungeon/`

### 2.1 启动入口

| 文件 | 路径 | 作用 |
|------|------|------|
| `DungeonApplication.java` | `com/dungeon/DungeonApplication.java` | `@SpringBootApplication` 注解的主类，包含 `main()` 方法。运行它启动整个 Spring Boot 后端。 |

### 2.2 实体类（Entity）

> 实体类 = 数据库表。每个 `@Entity` 类对应数据库中的一张表，JPA/Hibernate 自动建表和映射。
> 路径：`backend/src/main/java/com/dungeon/model/`

| 文件 | 数据库表名 | 存储内容 | 关键字段 |
|------|-----------|---------|---------|
| `User.java` | `USERS` | 用户账号 | `id`（自增主键）、`username`（唯一，最长32字符）、`passwordHash`（BCrypt加密，最长255字符）、`createdAt`（注册时间） |
| `SaveData.java` | `SAVE_DATA` | **大厅永久存档**（每个用户一条） | `user`（一对一关联User）、`gems`（宝石数）、`upgradesJson`（5项升级等级JSON）、`statsJson`（总场次/总击杀/最佳楼层JSON）、`updatedAt` |
| `RunSaveData.java` | `RUN_SAVE_DATA` | **单局中途存档**（每个用户一条） | `user`（一对一关联User）、`hasRun`（是否有进行中的对局）、`floorNum`（当前楼层）、`kills`（本局击杀）、`coins`（本局金币）、`gemsThisRun`（本局获得宝石）、`playerJson`（玩家完整状态JSON）、`roomsJson`（房间布局JSON）、`currentRoomIndex`、`updatedAt` |

**SaveData 的 upgradesJson 默认值：**
```json
{"maxHp":0, "maxShield":0, "maxEnergy":0, "cooldownReduct":0, "chestQuality":0}
```

**SaveData 的 statsJson 默认值：**
```json
{"totalRuns":0, "totalKills":0, "bestFloor":0}
```

### 2.3 DTO 数据传输对象

> DTO（Data Transfer Object）= 前后端之间传数据的"快递包裹"，不是数据库表，只是临时传输载体。
> 路径：`backend/src/main/java/com/dungeon/dto/`

| 文件 | 方向 | 包含字段 | 说明 |
|------|------|---------|------|
| `LoginRequest.java` | 前端→后端 | `@NotBlank @Size(min=3,max=32) username`、`@NotBlank @Size(min=4,max=64) password` | 登录请求 |
| `RegisterRequest.java` | 前端→后端 | 同上 | 注册请求 |
| `AuthResponse.java` | 后端→前端 | `token`（JWT令牌）、`username` | 登录/注册成功后的返回 |
| `SaveDataDTO.java` | 双向 | `gems`、`upgradesJson`、`statsJson` | 大厅存档传输 |
| `RunSaveDataDTO.java` | 双向 | `hasRun`、`floorNum`、`kills`、`coins`、`gemsThisRun`、`playerJson`、`roomsJson`、`currentRoomIndex` | 单局存档传输 |

### 2.4 Repository 数据库操作

> Repository = 和数据库交互的接口。继承 `JpaRepository`，不需要写 SQL，Spring Data JPA 自动生成增删改查。
> 路径：`backend/src/main/java/com/dungeon/repository/`

| 文件 | 自定义方法 | 说明 |
|------|-----------|------|
| `UserRepository.java` | `findByUsername(String username)` → `Optional<User>` | 按用户名查找用户 |
| | `existsByUsername(String username)` → `boolean` | 检查用户名是否已存在 |
| `SaveDataRepository.java` | `findByUserId(Long userId)` → `Optional<SaveData>` | 按用户ID查找大厅存档 |
| `RunSaveDataRepository.java` | `findByUserId(Long userId)` → `Optional<RunSaveData>` | 按用户ID查找单局存档 |

### 2.5 Service 业务逻辑

> Service = 核心业务逻辑层。Controller 调用 Service，Service 调用 Repository。
> 路径：`backend/src/main/java/com/dungeon/service/`

| 文件 | 核心方法 | 业务逻辑 |
|------|---------|---------|
| `JwtService.java` | `generateToken(username)` | 用 HMAC-SHA 密钥 + 用户名生成 JWT Token，有效期24小时 |
| | `extractUsername(token)` | 从 Token 中解析出用户名 |
| | `validateToken(token)` | 验证 Token 是否有效（签名正确 + 未过期） |
| `AuthService.java` | `register(username, password)` | ①检查用户名是否重复 ②BCrypt加密密码 ③保存用户到数据库 ④返回JWT Token |
| | `login(username, password)` | ①查找用户 ②BCrypt验证密码 ③返回JWT Token |
| `SaveService.java` | `loadSave(User)` | 读取大厅存档，首次访问自动创建默认存档 |
| | `saveSave(User, DTO)` | 更新大厅存档（宝石数、升级等级、战绩统计），带 `@Transactional` 事务 |
| | `loadRunSave(User)` | 读取单局中途存档，首次访问自动创建空记录 |
| | `saveRunSave(User, DTO)` | 更新单局存档（楼层、击杀、玩家/房间状态JSON），带事务 |
| | `deleteRunSave(User)` | 删除单局存档（玩家死亡或通关后调用） |

### 2.6 Controller API 接口

> Controller = 对外暴露的 HTTP 接口。前端 `api.js` 调的就是这些 URL。
> 路径：`backend/src/main/java/com/dungeon/controller/`

| 文件 | 接口 | 方法 | 认证 | 说明 |
|------|------|------|------|------|
| `AuthController.java` | `/api/auth/register` | POST | 无 | 注册，接收 `@Valid` RegisterRequest，返回 AuthResponse |
| | `/api/auth/login` | POST | 无 | 登录，接收 `@Valid` LoginRequest，返回 AuthResponse |
| `SaveController.java` | `/api/save/load` | GET | Bearer Token | 加载大厅存档，返回 SaveDataDTO |
| | `/api/save/save` | POST | Bearer Token | 保存大厅存档，接收 SaveDataDTO |
| | `/api/save/run/load` | GET | Bearer Token | 加载单局存档，返回 RunSaveDataDTO |
| | `/api/save/run/save` | POST | Bearer Token | 保存单局存档，接收 RunSaveDataDTO |
| | `/api/save/run/delete` | POST | Bearer Token | 删除单局存档 |

**认证方式：** `SaveController` 第39-49行通过私有方法 `getUserFromToken(authHeader)` 从 HTTP Header `Authorization: Bearer <token>` 中提取 JWT 并解析出用户，不依赖 Spring Security Filter 链。

> 本章以追踪视角，展示每个 API 从前端发起请求 → 后端处理 → 数据库读写的完整流程，所有位置精确到文件和行号。

| ④ | `/api/save/save` | POST | 保存大厅存档 | 需 Token |

### 3.2 接口 ①：注册 — `POST /api/auth/register`

**功能：** 新建用户账号，返回 JWT Token 实现自动登录。

#### 调用链路追踪

```
前端 game.js 第 2657 行
  │  api.register(username, password)
  │
  ▼
前端 api.js 第 66-73 行
  │  POST /api/auth/register
  │  body: { username, password }   ← 对应 RegisterRequest DTO
  │
  ▼
后端 AuthController.java 第 22-25 行
  │  @PostMapping("/register")
  │  public register(@Valid @RequestBody RegisterRequest request)
  │  ① @Valid 校验：username 长度3-32，password 长度4-64（不满足则返回400）
  │  ② 调用 authService.register(request)
  │
  ▼
后端 AuthService.java 第 23-31 行
  │  public AuthResponse register(RegisterRequest request)
  │  ① 调用 userRepository.existsByUsername() 检查用户名是否已被占用
  │     ↓ 已存在 → throw RuntimeException("用户名已存在") → 返回400
  │  ② 调用 passwordEncoder.encode(password)  → BCrypt 哈希加密密码明文
  │  ③ new User(username, hash)                → 创建用户实体对象
  │  ④ userRepository.save(user)               → INSERT INTO users 写入数据库
  │  ⑤ jwtService.generateToken(username)      → 生成24小时有效期的JWT令牌
  │  ⑥ return new AuthResponse(token, username) → 封装响应
  │
  ▼
返回前端 → api.js 第 71 行
  │  this.setAuth(data.token, data.username)
  │  保存 token 到 localStorage（键名: dungeon_auth_token）
  │  保存 username 到 localStorage（键名: dungeon_auth_user）
```

| 步骤 | 文件 | 行号 | 做什么 |
|------|------|------|--------|
| 发起请求 | `game.js` | 2657 | `api.register(u, p)` |
| HTTP 封装 | `api.js` | 66-73 | `POST /api/auth/register`，JSON序列化 body |
| 接收请求 | `AuthController.java` | 22-25 | `@PostMapping("/register")`，`@Valid` 参数校验 |
| 业务逻辑 | `AuthService.java` | 23-31 | 查重 → BCrypt加密 → 保存用户 → 生成JWT |
| 查重 | `UserRepository.java` | — | `existsByUsername()` → `SELECT COUNT(*) FROM users WHERE username=?` |
| 保存用户 | `UserRepository.java` | — | `save(user)` → `INSERT INTO users (username, password_hash, created_at) VALUES (?, ?, ?)` |
| 生成令牌 | `JwtService.java` | 25-34 | HMAC-SHA 签名 JWT，sub=username，有效期24h |
| 返回前端 | `api.js` | 71-72 | `setAuth()` 存 token 到 localStorage |

### 3.3 接口 ②：登录 — `POST /api/auth/login`

**功能：** 验证用户名密码，返回 JWT Token。

#### 调用链路追踪

```
前端 game.js 第 2663 行
  │  api.login(username, password)
  │
  ▼
前端 api.js 第 76-83 行
  │  POST /api/auth/login
  │  body: { username, password }   ← 对应 LoginRequest DTO
  │
  ▼
后端 AuthController.java 第 27-30 行
  │  @PostMapping("/login")
  │  public login(@Valid @RequestBody LoginRequest request)
  │  ① @Valid 校验
  │  ② 调用 authService.login(request)
  │
  ▼
后端 AuthService.java 第 34-42 行
  │  public AuthResponse login(LoginRequest request)
  │  ① userRepository.findByUsername(username)
  │     ↓ 找不到 → throw RuntimeException("用户不存在")
  │  ② passwordEncoder.matches(password, user.getPasswordHash())
  │     ↓ 不匹配 → throw RuntimeException("密码错误")
  │  ③ jwtService.generateToken(username)
  │  ④ return new AuthResponse(token, username)
```

| 步骤 | 文件 | 行号 | 做什么 |
|------|------|------|--------|
| 发起请求 | `game.js` | 2663 | `api.login(u, p)` |
| HTTP 封装 | `api.js` | 76-83 | `POST /api/auth/login` |
| 接收请求 | `AuthController.java` | 27-30 | `@PostMapping("/login")` |
| 业务逻辑 | `AuthService.java` | 34-42 | 查用户 → BCrypt验证密码 → 生成JWT |
| 查询用户 | `UserRepository.java` | — | `findByUsername()` → `SELECT * FROM users WHERE username=?` |
| 密码验证 | `AuthService.java` | 37 | `passwordEncoder.matches()` → BCrypt比对 |
| 返回前端 | `api.js` | 81-82 | `setAuth()` 存 token |

### 3.4 接口 ③：读取大厅存档 — `GET /api/save/load`

```
前端 game.js 第 56 行 / 第 2724 行
  │  const cloud = await api.loadSave()
  │  调用时机：① 页面初始化时（第56行）② 进入大厅时（第2724行）
  │
  ▼
前端 api.js 第 88-90 行
  │  GET /api/save/load
  │  header: Authorization: Bearer <token>   ← 自动附加
  │
  ▼
后端 SaveController.java 第 59-63 行
  │  @GetMapping("/load")
  │  public load(@RequestHeader("Authorization") String authHeader)
  │  ① getUserFromToken(authHeader) → 解析Token获取User对象（第39-49行）
  │  ② saveService.loadSave(user)
  │
  ▼
后端 SaveService.java 第 39-59 行
  │  public SaveDataDTO loadSave(User user)
  │  ① saveDataRepository.findByUserId(user.getId())
  │     ↓ 无存档 → 创建默认存档（gems=0, upgrades全0, stats全0）
  │  ② 将 SaveData 实体 → 转换为 SaveDataDTO
  │  ③ return dto
  │
  ▼

  │  findByUserId() → SELECT * FROM save_data WHERE user_id=?
  │
  ▼
返回前端 → game.js 第 57-68 行
  │  合并策略：cloud 和 localStorage 取较新的值
  │  如果云端数据较新，更新本地 globalSave
```

| 步骤 | 文件 | 行号 | 做什么 |
|------|------|------|--------|
| 发起请求（初始化） | `game.js` | 56 | `api.loadSave()` |
| 发起请求（进大厅） | `game.js` | 2724 | `api.loadSave()` |
| HTTP 封装 | `api.js` | 88-90 | `GET /api/save/load`，自动带 Token |
| 接收请求 | `SaveController.java` | 59-63 | `@GetMapping("/load")` |
| Token 解析 | `SaveController.java` | 39-49 | `getUserFromToken()`：去"Bearer "前缀→验证JWT→提取用户名→查用户 |
| 业务逻辑 | `SaveService.java` | 39-59 | 查数据库→无则创建默认→转DTO |
| 数据库查询 | `SaveDataRepository.java` | — | `findByUserId()` → `SELECT * FROM save_data WHERE user_id=?` |
| 合并存档 | `game.js` | 57-68 | localStorage 与 cloud 取较新值 |

### 3.5 接口 ④：保存大厅存档 — `POST /api/save/save`

**功能：** 将大厅存档同步到云端（宝石消费后、升级购买后、战绩更新后）。

#### 调用链路追踪

```
前端 game.js 第 79-84 行
  │  await api.saveSave({ gems, upgradesJson, statsJson })
  │  调用时机：每次 saveToDisk() 时同步到云端
  │
  ▼
前端 api.js 第 95-100 行
  │  POST /api/save/save
  │  header: Authorization: Bearer <token>
  │  body: { gems, upgradesJson, statsJson }   ← 对应 SaveDataDTO
  │
  ▼
后端 SaveController.java 第 72-78 行
  │  @PostMapping("/save")
  │  public save(@RequestHeader("Authorization") String authHeader,
  │              @RequestBody SaveDataDTO dto)
  │  ① getUserFromToken(authHeader) → 解析Token获取User
  │  ② saveService.saveSave(user, dto)
  │
  ▼
后端 SaveService.java 第 66-81 行
  │  @Transactional
  │  public void saveSave(User user, SaveDataDTO dto)
  │  ① saveDataRepository.findByUserId(user.getId())
  │     ↓ 无存档 → 新建实体对象
  │  ② save.setGems(dto.getGems())
  │     save.setUpgradesJson(dto.getUpgradesJson())
  │     save.setStatsJson(dto.getStatsJson())
  │     save.setUpdatedAt(LocalDateTime.now())
  │  ③ saveDataRepository.save(save)  → INSERT 或 UPDATE
  │
  ▼
返回 {"success": true}
```

| 步骤 | 文件 | 行号 | 做什么 |
|------|------|------|--------|
| 发起请求 | `game.js` | 79-84 | `api.saveSave({ gems, upgradesJson, statsJson })` |
| HTTP 封装 | `api.js` | 95-100 | `POST /api/save/save` |
| 接收请求 | `SaveController.java` | 72-78 | `@PostMapping("/save")` |
| 业务逻辑 | `SaveService.java` | 66-81 | `@Transactional` 事务写数据库 |
| 数据库写入 | `SaveDataRepository.java` | — | `save()` → JPA 自动判断 INSERT 或 UPDATE |

### 3.6 接口 ⑤：读取单局存档 — `GET /api/save/run/load`

```
前端 game.js 第 2442 行
  │  const data = await api.loadRunSave()
  │  调用时机：loadRunState() 恢复存档时
  │
  ▼
前端 api.js 第 105-107 行
  │  GET /api/save/run/load
```

### 3.10 认证机制

整个认证流程不依赖 Spring Security 的 Filter 链，而是**手动在 Controller 层处理**：

```
   前端发送请求（api.js 第 46 行自动带 Bearer Token）
       │
       ▼
   SaveController.getUserFromToken()  第 39-49 行
       │
       ├─ ① 检查 Authorization header 是否存在且以 "Bearer " 开头
       ├─ ② 去掉 "Bearer " 前缀 → authHeader.substring(7)   第 43 行
       ├─ ③ jwtService.validateToken(token)                  第 44 行
       │      └─ JwtService.java 第 40-47 行：解析 + 验证签名 + 检查过期
       ├─ ④ jwtService.extractUsername(token)                第 47 行
       │      └─ JwtService.java 第 36-38 行：从 JWT payload 提取 subject
       └─ ⑤ authService.getUserByUsername(username)          第 48 行
              └─ AuthService.java 第 44-47 行：查数据库返回 User 对象
```

**SecurityConfig.java（第 1 行起）：**
- CSRF → 关闭
- `/api/auth/**` → 所有人可访问（第 15 行 `permitAll()`）
- `/h2-console/**` → 所有人可访问
- 其余所有路径 → 所有人可访问（不拦截，认证逻辑在 Controller 手动处理）

**JwtService.java（第 1-56 行）：**
- `generateToken()` 第 25-34 行 → HMAC-SHA 签名，sub=username，24h有效期
- `extractUsername()` 第 36-38 行 → 解析 token 提取用户名
- `validateToken()` 第 40-47 行 → 验证签名 + 检查过期

**SecurityConfig.java 中的 BCrypt Bean：**
- `passwordEncoder()` → `BCryptPasswordEncoder`，用于 AuthService 加密/验证密码

| 组件 | 说明 |

| `showShop()` | 4件商品：治疗30金币/护盾20金币/能量25金币/随机武器50金币 |
| `showSettlement(win)` | 结算宝石（击杀×2+楼层×10+50通关奖）+ 更新战绩 + 删单局存档 |
